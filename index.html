Guía de integración

Ejemplos de código

Integra la API en tu lenguaje favorito en minutos.

Endpoint

POST /api/v1/consultas

Body: {"token": "TU_API_KEY", "ruc": "20601138572"}

Endpoint

POST /api/v1/consultas-dni

Body: {"token": "TU_API_KEY", "dni": "12345678"}

Endpoint

GET /api/v1/tipo-cambio

Query params: ?token=TU_API_KEY&fecha=2025-01-15 (fecha es opcional, formato YYYY-MM-DD)

O como body JSON: {"token": "TU_API_KEY", "fecha": "2026-04-17"}

Endpoint

POST /api/v1/cpe/consultar

Body: {"token":"TU_API_KEY","ruc":"20601138572","tipo":"01","serie":"F001","correlativo":"1234","fecha":"2026-01-15","monto":1180.00}

tipo: 01=Factura · 03=Boleta · 07=Nota de crédito · 08=Nota de débito

consulta.php 
<?php

// Configura tu API Key
$apiKey = "tu_api_key";
$ruc    = "20601138572";
$ch = curl_init("https://api.consultaperuapi.com/api/v1/consultas");

    CURLOPT_RETURNTRANSFER => true,
    CURLOPT_POST           => true,

    CURLOPT_POSTFIELDS     => json_encode(["token" => $apiKey, "ruc" => $ruc])
]);

$body = curl_exec($ch);
$data = json_decode($body);

echo $data->razonSocial;  // "EMPRESA EJEMPLO S.A.C."
echo $data->estado;       // "ACTIVO"

validar_cpe.php 
<?php

$apiKey = "tu_api_key";
$payload = [
    "token"      => $apiKey,
    "ruc"        => "20601138572",
    "tipo"       => "01",
    "serie"      => "F001",
    "correlativo"=> "1234",
    "fecha"      => "2026-01-15",
    "monto"      => 1180.00
];

$ch = curl_init("https://api.consultaperuapi.com/api/v1/cpe/consultar");
curl_setopt_array($ch, [
    CURLOPT_RETURNTRANSFER => true,
    CURLOPT_POST           => true,
    CURLOPT_HTTPHEADER     => ["Content-Type: application/json"],
    CURLOPT_POSTFIELDS     => json_encode($payload)
]);

$data = json_decode(curl_exec($ch));

echo $data->valido ? "Válido" : "No válido";
echo $data->estado;          // "ACEPTADO"
echo $data->descripcion_tipo; // "Factura"

Campos de la respuesta

ruc

Número de RUC del contribuyente (11 dígitos)

razonSocial

Nombre o razón social registrada en SUNAT

estado

Estado del contribuyente: ACTIVO, BAJA, etc.

condicion

Condición de domicilio: HABIDO, NO HABIDO, etc.

direccion

Domicilio fiscal registrado

ubigeo

Ubicación geográfica: Departamento, Provincia, Distrito

Campos de la respuesta

fecha

Fecha del tipo de cambio (YYYY-MM-DD)

moneda

Código de moneda (siempre "USD")

compra

Tipo de cambio de compra (número)

venta

Tipo de cambio de venta (número)

fuente

Fuente del dato (siempre "SBS")

timestamp_actualizacion

Timestamp de última actualización (ISO 8601)

Campos de la respuesta

valido

true si el comprobante existe y es válido en SUNAT

estado

ACEPTADO, ANULADO, NO EXISTE, AUTORIZADO o NO AUTORIZADO

descripcion_tipo

Factura, Boleta de venta, Nota de crédito o Nota de débito

monto_total

Monto total del comprobante (número)

fecha_emision

Fecha de emisión registrada en SUNAT (YYYY-MM-DD)

estado_contribuyente

Estado del emisor: ACTIVO, BAJA DEFINITIVA, etc.

condicion_domicilio

Condición de domicilio del emisor: HABIDO, NO HALLADO, etc.

fuente

Siempre "SUNAT"

Campos de la respuesta

dni

Número de DNI consultado (8 dígitos)

nombres

Nombres de la persona

apellidoPaterno

Apellido paterno

apellidoMaterno

Apellido materno

nombreCompleto

Nombre completo formateado

direccion

Dirección del domicilio fiscal (puede ser null)

fuente

Fuente de datos: padron_local o callback
